# 高铁无线智能化运营-界面整改执行说明

## 全局
- 目标：从“科技感组件堆叠”收敛到“可信赖的展会级运营系统”。
- 删除或弱化所有不必要的强描边、亮角和发光外框，主容器保留结构感，次级容器只保留轻边界。
- 统一信息层级：
  - 一级：系统标题
  - 二级：页面主模块标题
  - 三级：区块标题
  - 四级：标签、表头、说明
  - 数值：仅关键指标放大，其余维持克制
- 统一颜色语义：
  - 青色：当前焦点、关键收益
  - 绿色：正常
  - 黄色：预警
  - 红色：仅当前主问题
  - 其余信息回归中性色
- 背景与网格降噪，减少“AI 大屏模板感”。

## P0 线路态势
- 保持“地图主导”不变，继续让左侧地图成为第一视觉。
- 右侧运行总览、线路排序、车型对比改为更轻量的运营看板，减少每块都像发光卡片。
- 地图站点标注做主次层级：
  - 枢纽站更亮
  - 普通站更轻
  - 避免整图字重一致
- 地图浮层、图例、线路问题卡尺寸再压缩，给线路与站点更多可视面积。

## P1 感知洞察
- 保持“告警主导”逻辑不变。
- 地图与时间轴整合为同一个分析舞台，消除割裂感与中间空白。
- 左侧地图信息层级调整：
  - 线路与连续采样是主层
  - 站点与告警是第二层
  - 图例与详情卡是第三层
- 右侧三块面板统一收轻，降低描边和发光噪音。
- 当前主告警保持清晰突出，其余告警弱提示但不消失。

## P2 策略保障
- 保障策略区继续保持“轻量按钮条 + 参数摘要带”。
- 删除重复信息：
  - 不再重复显示“当前策略/保障前”等已被按钮或状态表达的信息
  - 策略范围收敛为“18个APP / 查看APP明细”
- 双手机对比升级成真正的主视觉：
  - 更像终端设备，而不是普通容器
  - 保留录屏替换位，但不写解释型文字
- 关键业务体验差异改成更像收益看板，不像普通表格。
- 流量使用作为新增收益指标保留，用于表达体验改善带来的使用激发。

## 执行原则
- 先改共享样式，再改页面局部结构。
- 所有调整以“更克制、更清晰、更像成品”为目标。
- 不新增无必要交互，不推翻已确认的信息结构。
