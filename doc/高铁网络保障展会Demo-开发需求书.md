## 1. 项目目标

本 Demo 用于展示“无线智能板”在高铁网络保障场景中的四类能力：

1. `P0 虚拟路测`  
   智能板具备业务级、5 秒级感知能力，可以在列车瞬时冲击时识别用户真实体验问题，而传统 PRS 只能看到 15 分钟聚合后的“看似正常”结果。

2. `P1 感知洞察`  
   智能板能够识别关键业务指标、质差次数、异常区段，并给出问题根因解释。

3. `P2 分层分级保障`  
   可以通过调整不同业务、不同用户的 `5QI / RFSP` 策略，实现对 VIP 用户和关键业务的优先调度与体验提升。

4. `P3 投诉处理闭环`  
   可以按匿名化投诉或告警进行 5 秒级回溯，联动多指标证据链，输出智能问题分析和闭环建议。

这个产品的核心故事线必须固定为：

`看得见 -> 看得准 -> 保得住 -> 查得清`

## 11. 技术实现要求

### 11.1 状态管理

当前项目可继续使用轻量级 React 本地状态，不强制引入 Zustand、Redux 等全局状态库。

建议状态划分：

- `App.tsx`
  - 当前屏 `currentScreen`
  - 当前场景 `sceneIndexByScreen`
  - 高亮开关 `highlightMode`
  - 自动播放开关 `autoPlay`
  - 重置版本号 `resetNonce`

- 各页面组件内部
  - 当前选中列车
  - 当前选中问题
  - 当前选中投诉
  - 当前回放帧
  - 当前业务类型

### 11.2 场景引擎

所有页面都必须围绕“讲解场景”组织，而不是围绕“通用系统功能”组织。

场景引擎需要具备以下能力：

- 每个屏幕定义多个场景
- 每个场景有标题、摘要、讲解提示
- 每个场景绑定默认焦点：
  - 线路
  - 车次
  - 区段
  - 问题
  - 投诉
  - 帧索引

### 11.3 地图实现

优先能力：

- 使用高德地图 JS API
- 支持线路 Polyline
- 支持站点 Marker
- 支持风险区段高亮
- 支持列车动态位置

兜底能力：

- 若高德地图加载失败，则启用本地静态轨道底图
- 本地模式下依然要显示：
  - 线路
  - 站点
  - 区段
  - 列车点位
  - 当前高亮

### 11.4 响应式实现

必须适配：

- 1920 x 1080
- 1366 x 768
- 笔记本全屏浏览器

响应式原则：

- 优先保持结构完整
- 次要信息允许折叠
- 右侧讲解面板在小屏下允许收窄到 `300px`
- 图表高度可以随屏幕压缩，但不可溢出

### 11.5 性能要求

- 首屏加载时间小于 `3 秒`
- 切屏时间小于 `1 秒`
- 时间轴步进响应小于 `200ms`
- 纯前端不依赖远程接口
- 所有图表和地图在没有数据时必须优雅降级

## 17. 给开发 Agent 的最终执行原则

请严格遵守以下原则：

1. 优先做“展会稳定可演示”，而不是“系统完备”

1. 保留主故事线
2. 保留大屏视觉冲击力
3. 保留四屏完整度
4. 保留讲解员操作能力
5. 最后再裁剪次要图表或辅助文案
