# 高铁无线智能化运营 字体与色彩规范

## 1. 规范来源

本规范参考 AntV 设计价值观页面中的三条原则，并转化为本项目的大屏设计规则：

- `信任感`
  - 信息表达必须准确、克制，不堆叠无效装饰，不制造“像演示稿”的感觉。
- `轻松感`
  - 信息层级必须清楚，文本、图形、标签与背景的对比度必须足够，用户要能快速读懂。
- `意义感`
  - 页面必须突出关键数据、关键异常和关键动作，让用户在几秒内抓住重点。

参考来源：
- [AntV 设计价值观 | 设计价值观](https://antv.antgroup.com/specification/principles/basic)

---

## 2. 本项目的视觉目标

本项目不是普通后台，不是营销海报，而是“正式运行中的高铁无线智能化运营系统”。

所以视觉上遵循以下原则：

- 头部要稳，不要花
- 标题要短，不要口号化
- 说明文字要少，不要像 PPT
- 数据要大，要有优先级
- 颜色只服务于状态和重点，不做无意义彩色堆叠

## 3. 字体栈规范

### 3.1 中文主字体

全局中文字体：

```css
'Alibaba PuHuiTi 3.0', 'PingFang SC', 'Microsoft YaHei UI', 'Microsoft YaHei', sans-serif
```

选择原因：

- 中文显示稳定
- 商务系统感强
- 在 Windows 展会设备上兼容性更好

### 3.2 数字与指标字体

大数字和关键指标使用：

```css
'DIN Alternate', 'Bahnschrift', 'Rajdhani', 'Alibaba PuHuiTi 3.0', sans-serif
```

选择原因：

- 数字辨识度高
- 横向密度适合大屏
- 适合速率、时延、评分、数量等运营数据

## 4. 字体层级规范

### 4.1 一级标题

用途：

- 系统头部标题
- 页面主标题

规范：

- 字号：`28px ~ 34px`
- 字重：`700`
- 行高：`1.18`
- 颜色：`#EFF7FF`

当前 token：

```css
--font-title-1: clamp(28px, 2vw, 34px);
```

### 4.2 二级标题

用途：

- 各页面核心模块标题
- 大卡片标题

规范：

- 字号：`22px ~ 28px`
- 字重：`700`
- 行高：`1.18`
- 颜色：`#EFF7FF`

当前 token：

```css
--font-title-2: clamp(22px, 1.5vw, 28px);
```

### 4.3 三级标题

用途：

- 面板标题
- 子模块标题

规范：

- 字号：`18px`
- 字重：`600`
- 行高：`1.3`
- 颜色：`#EFF7FF`

当前 token：

```css
--font-title-3: 18px;
```

### 4.4 正文

用途：

- 模块说明
- 证据链说明
- 闭环建议说明

规范：

- 正文大号：`16px`
- 正文标准：`14px`
- 行高：`1.6`
- 颜色：`rgba(224, 242, 255, 0.78)`

当前 token：

```css
--font-body-lg: 16px;
--font-body: 14px;
--text-secondary: rgba(224, 242, 255, 0.78);
```

### 4.5 说明文字 / 标签文字

用途：

- 指标说明
- 次级标注
- overline 小标签

规范：

- Caption：`12px`
- Overline：`11px`
- 行高：`1.4 ~ 1.6`
- 颜色：
  - 说明文字：`rgba(224, 242, 255, 0.78)`
  - 次级标注：`rgba(224, 242, 255, 0.56)`

当前 token：

```css
--font-caption: 12px;
--font-overline: 11px;
--text-tertiary: rgba(224, 242, 255, 0.56);
```

## 5. 数据字体规范

### 5.1 超大数据

用途：

- 总量 KPI
- 核心评分
- 强调型指标

规范：

- 字号：`36px ~ 48px`
- 字重：`700`
- 数字字体：`DIN Alternate / Bahnschrift`
- 颜色：主白或状态色

当前 token：

```css
--font-data-xl: clamp(36px, 2.6vw, 48px);
```

### 5.2 大数据

用途：

- 重点指标卡
- 业务结果对比

规范：

- 字号：`28px`
- 字重：`700`

当前 token：

```css
--font-data-lg: 28px;
```

### 5.3 中数据

用途：

- 普通指标卡
- 地图右侧状态卡

规范：

- 字号：`22px`
- 字重：`600 ~ 700`

当前 token：

```css
--font-data-md: 22px;
```

## 7. 颜色使用规则

### 7.1 主白

用于：

- 标题
- 核心内容
- 当前重要文字

### 7.2 青色

用于：

- 信息型高亮
- 中性重点
- 地图线路、参考指标、普通激活态

### 7.3 橙色

用于：

- 当前操作焦点
- 当前高亮对象
- 页签激活
- 需要讲解者重点点击的位置

### 7.4 黄色

用于：

- 风险预警
- 待处理状态
- 信号或驻留劣化但未到严重级别

### 7.5 红色

用于：

- 严重异常
- 关键风险
- 投诉关键告警

### 7.6 绿色

用于：

- 已生效
- 已闭环
- 恢复正常

## 8. 页面中的标注颜色规则

### 8.1 一级标题

- 颜色：主白

### 8.2 二级标题

- 颜色：主白

### 8.3 面板说明

- 颜色：次级文字

### 8.4 数据标签

- 颜色：三级文字

### 8.5 数据数值

- 默认颜色：主白
- 风险：红色 / 黄色
- 改善：绿色
- 激活：橙色

---

## 9. 本项目执行规则

后续所有页面开发都按下面的统一约束：

1. 一级标题不超过 14 个汉字
2. 二级标题不超过 18 个汉字
3. 单段说明文不超过 2 行
4. 一个指标卡里，标签永远小于数值
5. 一个面板里，颜色主角最多 1 种
6. 橙色只能用来强调“当前焦点”
7. 红色只能用来表示“风险和异常”
8. 不允许同一块区域同时出现多种强对比颜色争夺注意力
