# 高铁无线智能化运营展会 Demo

## 项目简介

这是一个面向展会演示场景的纯前端大屏项目，用于展示无线智能板在高铁网络保障场景下的核心能力。  
项目不依赖后台系统，不接入真实生产链路，所有能力均通过本地 mock 数据、预设场景和可视化联动进行呈现，重点保证：

- 演示稳定
- 叙事清晰
- 视觉完整
- 后续易于替换真实数据

当前系统主线包含 4 个页面：

- `P0 线路态势`
  - 展示省域高铁线路总体态势、线路质量排序、车型对比
  - 支持线路高亮、风险线路识别、地图地名展示
- `P1 感知洞察`
  - 聚焦单条样板线路
  - 展示连续采样、告警打点、时间轴切片、速率与业务感知指标
- `P2 策略保障`
  - 展示针对 VIP 用户和关键业务的分层分级保障能力
  - 体现 5QI / RFSP 等理论可执行策略的体验收益
- `P3 投诉回溯`
  - 从匿名化投诉进入问题回溯
  - 联动地图、时间、指标和闭环建议

## 设计目标

本项目不是后台系统原型，而是“可讲、可演、可替换数据”的高完成度展会 Demo。

设计原则如下：

- 页面表现尽量接近真实运营系统，而不是概念页
- 地图与指标联动优先，减少冗余说明文字
- 所有关键数据结构都支持后续替换为真实数据
- 允许使用 mock 数据强化故事线，但不破坏整体可信度

## 运行方式

### 一键启动

在项目根目录执行：

```bat
start_demo.bat
```

脚本会自动完成以下步骤：

1. 检查依赖
2. 执行构建
3. 启动本地预览服务
4. 自动打开浏览器

默认访问地址：

```text
http://127.0.0.1:4173
```

### 一键关闭

```bat
stop_demo.bat
```

### 手动开发

```bash
npm install
npm run dev
```

### 手动构建预览

```bash
npm run build
npm run preview -- --host 127.0.0.1 --port 4173
```

## 环境变量

项目支持通过 `.env` 配置高德地图参数。

示例见：

- [D:\Codex\crh\.env.example](D:\Codex\crh\.env.example)


## 数据说明

当前项目的数据来源分为 3 类：

- 静态 mock 数据
  - 页面默认演示数据
  - 位于 `src/mock/`
- 地图线路数据
  - 位于 `public/`
  - 包含裁剪后的高铁线路 GeoJSON
- 预处理脚本
  - 位于 `scripts/`
  - 用于从大体量 GeoJSON 中提取可用于页面展示的子集

当前实现中，页面已经尽量按“真实数据可替换”方式设计，后续如果你提供真实数据，通常只需要替换：

- `src/mock/` 下的指标数据
- `public/` 下的线路或点位数据

而不需要重写页面结构。

## 页面说明

### P0 线路态势

核心目标：

- 从全局视角展示省域高铁网络态势
- 快速识别风险线路和重点线路
- 用地图 + 排行 + 车型对比构成完整故事线

当前能力：

- 高德暗色底图 / SVG 兜底
- 线路高亮联动
- 地名标注
- 线路质量排序
- 智能板覆盖与列次监测概览

### P1 感知洞察

核心目标：

- 从单条样板线路深钻问题
- 展示线路连续采样、问题切片和业务感知

当前能力：

- 单线路走廊地图
- 1000 米级连续采样打点
- 5 个主要问题点高亮
- 2 小时 / 24 个五分钟切片时间轴
- 告警点、地图详情、右侧指标联动
- 站点名称标注

### P2 策略保障

核心目标：

- 演示分层分级保障思路
- 展示不同业务在保障前后的收益变化

当前能力：

- 保障场景卡片
- 前后对比
- 重点业务收益展示

说明：

- 当前 P2 无地图主舞台

### P3 投诉回溯

核心目标：

- 从投诉出发进行问题回放
- 形成从投诉到建议的闭环展示

当前能力：

- 投诉列表
- 地图联动
- 时间回溯
- 问题分析
- 闭环建议
- 地图地名 / 站名标注

## 已知边界

当前项目有明确边界，属于展会版演示系统：

- 不接真实网管后台
- 不接真实投诉系统
- 不做真实工单流转
- 不做真实参数下发
- 不依赖实时 12306 接口
- “智能分析”与“闭环建议”当前为规则和场景模板驱动

## 推荐演示顺序

推荐讲解路径：

1. `P0 线路态势`
   - 先讲全局线路态势和省域质量分布
2. `P1 感知洞察`
   - 再讲单线路连续采样和告警切片
3. `P2 策略保障`
   - 再讲策略如何改善关键业务体验
4. `P3 投诉回溯`
   - 最后讲如何从投诉反查网络问题并给出处置建议
