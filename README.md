# 无线智能板高铁网络保障展会 Demo

## 项目说明

这是一个纯前端的展会演示 Demo，用来展示无线智能板在高铁场景下的四类能力：

1. `P0 虚拟路测`
   - 对比传统 `PRS 15 分钟` 和智能板 `5 秒级` 体验感知
   - 展示全网线路、在途列车、风险区段、车型差异

2. `P1 感知洞察`
   - 展示业务级 KQI、质差次数、线路热区和 8 类典型问题模板
   - 点击问题即可联动地图、指标和智能分析结论

3. `P2 分层分级保障`
   - 演示 VIP 用户在短视频、游戏、微信上传、下载业务上的前后对比
   - 通过 `5QI / RFSP` 和调度优先级变化展示“理论上可实施”的策略价值

4. `P3 投诉处理闭环`
   - 从匿名化投诉进入 5 秒级指标回溯
   - 联动地图位置、多指标时间轴、智能分析结论和闭环建议

这个项目**不对接后台**，所有数据都来自本地 mock 和预置场景，目标是保证展会展示稳定、故事完整、视觉冲击力足够强。

## 技术方案

- 前端框架：`React 19 + TypeScript + Vite`
- 地图能力：高德地图 JS API，加载失败时自动降级到离线轨道态势图
- 数据模式：本地静态 mock 数据，不依赖任何后端 API
- 演示模式：以讲解员手动演示为主，带轻量轮播和分镜控制台

## 一键启动

双击或在当前目录运行：

```bat
start_demo.bat
```

脚本会自动执行以下流程：

1. 检查 `node_modules`
2. 如有需要执行 `npm install`
3. 执行 `npm run build`
4. 启动 `vite preview`
5. 自动打开浏览器到 `http://127.0.0.1:4173`

说明：

- 选择 `preview` 而不是 `dev`，是为了展会环境更稳定
- 启动脚本会把进程 PID 写到 `.runtime\preview.pid`
- 如果已经启动，再次运行会直接提示并打开浏览器

## 一键关闭

双击或在当前目录运行：

```bat
stop_demo.bat
```

脚本会根据 `.runtime\preview.pid` 结束整个预览进程树，并删除 PID 文件。

## 手动运行

如果你想手动调试：

```bash
npm install
npm run dev
```

如果你想手动预览构建结果：

```bash
npm run build
npm run preview -- --host 127.0.0.1 --port 4173
```

## 展会演示建议

推荐讲解顺序：

1. 先从 `P0` 讲“为什么 5 秒级感知比 15 分钟均值更有价值”
2. 再进 `P1` 讲“智能板不仅能看到，还能分析出原因”
3. 进入 `P2` 讲“针对 VIP 和关键业务可做差异化保障”
4. 最后在 `P3` 用投诉闭环收尾，强调“能从用户反馈反查网络问题并给出建议”

## 当前边界

- 不接真实投诉系统
- 不接真实网管后台
- 不做真实工单流转
- 不做真实网络参数下发
- 所有“智能分析”和“闭环建议”都基于预设规则与案例模板

这正是为了保证展会 Demo 稳定、清楚、易讲。
