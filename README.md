# 高铁无线智能化运营 Demo

## 项目简介
这是一个面向展会讲解场景的纯前端大屏 Demo，用于展示无线智能板在高铁网络保障中的核心能力。项目不依赖后台接口，所有页面均由本地 mock 数据、地图轨迹数据和可视化联动驱动，目标是保证：

- 演示稳定
- 叙事清晰
- 视觉完整
- 后续易于替换真实数据

当前版本保留 3 个核心页面：

- `P0 线路态势`
- `P1 感知洞察`
- `P2 策略保障`

## 页面说明

### P0 线路态势
用于展示省域高铁线路总览、重点风险线路、线路质量排序和车型差异。

核心能力：
- 省域线路地图与排序联动
- 重点线路高亮
- 路线质量排序
- 车型信号损耗对比

### P1 感知洞察
用于深钻单条样板线，展示连续采样、告警点位和时间切片联动。

核心能力：
- 单线路放大地图
- 连续采样点位展示
- 5 个主要告警点联动
- 时间轴五分钟切片
- 右侧速率、业务、问题列表联动

### P2 策略保障
用于展示分层分级保障策略的直观收益。

核心能力：
- 轻量策略选择区
- 参数摘要带
- 双手机对比区
- 关键业务体验差异表
- 策略前后状态切换

## 启动方式

### 一键启动
在项目根目录执行：

```bat
start_demo.bat
```

默认访问地址：

```text
http://127.0.0.1:4173
```

### 一键关闭

```bat
stop_demo.bat
```

### 手动开发

```bash
npm install
npm run dev
```

### 手动构建预览

```bash
npm run build
npm run preview -- --host 127.0.0.1 --port 4173
```

## 环境变量
参考文件：

- [D:\Codex\crh\.env.example](D:\Codex\crh\.env.example)

常用配置：

```env
VITE_AMAP_KEY=your_amap_key
VITE_AMAP_SECURITY_CODE=your_amap_security_code
```

说明：
- 高德地图加载失败时，部分页面会自动回退到本地 SVG 或离线渲染模式
- `.env` 不会提交到仓库

## 数据说明
当前项目主要使用三类数据：

- 页面 mock 数据
- 地图与线路 GeoJSON 数据
- 预处理脚本生成的数据切片

后续如果需要替换为真实数据，通常只需要更新：

- `src/mock/`
- `public/`

而不需要重写页面结构。

## 已知边界
这是展会版演示系统，不追求真实生产链路，当前边界如下：

- 不接真实网管后台
- 不接真实投诉系统
- 不做真实参数下发
- 不做真实工单流转
- 不依赖实时 12306 接口
- 智能分析和闭环建议以规则与 mock 数据驱动
